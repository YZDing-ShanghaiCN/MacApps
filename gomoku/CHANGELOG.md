# Changelog

## [0.3.3] - 2026-07-22

- Web 人机页面新增可折叠 AI 调试面板，可查看搜索深度、节点、耗时、VCF 和预算状态，并复制或下载可复现问题局面；Pygame 新增 `Export Position` 按钮，使用同一 JSON 格式。
- 问题局面导出包含应用版本、完整棋盘与落子历史、当前玩家、模式、难度、NormalAI 全量配置和最后搜索统计，预留备注与期望落点字段，便于直接转为回归测试。
- 增加固定种子的随机差分测试，反复落子与撤销后逐步核对增量评估、全盘棋型、候选顺序、邻域集合、空位数量和 Zobrist 哈希。
- VCF 改为保留结构化证明链；证明中的进攻点、强制应点和最终五连点全部强制进入防守验证，不受普通防守候选数量限制。
- VCF 时间预算根据冲四、活三、跳三、眠三和多重威胁的紧迫程度动态缩放，并在搜索统计中记录分配预算和实际 VCF 耗时。
- 配置竞技场扩充为五组固定开局和双方换色，增加完整棋谱、先后手胜率、预算停止率、VCF 命中率、得分率、Elo 差估计、95% 区间与 JSON 报告输出。
- 将应用元数据和 PWA 静态资源缓存版本统一更新为 0.3.3；困难难度继续保留强化学习自博弈路线，本版不加入模型训练代码。

## [0.3.2] - 2026-07-20

- 为 SimpleAI 增加全棋盘一步获胜与必须阻挡保底，能够处理 `XXX_X`、`XX_XX` 等带间隔的直接五连威胁，同时保留其余简单难度连续棋串策略。
- NormalAI 的 SearchPosition 增加可撤销增量棋型状态；落子和撤销只更新经过该点的棋线，候选生成与静态评估共享结果，减少搜索节点的重复全盘扫描。
- 修正候选排序未采用独立防守棋型分值的问题，使 `defense_pattern_scores` 和 `defense_factor` 同时作用于走法排序与叶子局面评估。
- 增加受独立时间预算和候选上限约束的防守型 VCF：先验证对手连续冲四证明，再将根搜索限制到能够破坏该证明的安全落点。
- 新增可扩充的固定战术局面语料，覆盖间隔五连、棋盘边缘、双重威胁、防守 VCF、旋转和黑白对称性；增加基于固定节点预算、交换黑白和多开局的 NormalAI 配置对战工具。
- 记录困难难度采用独立强化学习自博弈路线的产品方向；0.3.2 不加入训练或未使用的模型代码，NormalAI 继续保持纯搜索。
- 将应用元数据和 PWA 静态资源缓存版本统一更新为 0.3.2。

## [0.3.1] - 2026-07-17

- 强化 NormalAI 搜索：加入 PVS、窄窗重搜、历史启发、杀手走法、威胁静态延伸的 stand-pat 处理，以及可选的确定性节点预算；超时仍只采用最后一个完整深度的结果。
- 增加受时间与深度限制的 VCF 连续冲四搜索；完整 VCT 继续留待实战反馈后单独开发。
- 将置换表改为组相联结构并增加代际替换、碰撞和替换统计；增加局面评估缓存、棋线识别缓存，以及 SearchPosition 候选邻域的增量维护。
- 分离进攻与防守棋型权重，修正双重威胁按独立棋形去重，并让中心位置奖励随对局阶段衰减；保留调参后的冲四 25,000、跳三 15,000 默认分值。
- Pygame 和 Web 的 AI 搜索改为后台执行，支持重开、悔棋、模式或难度切换时协作取消并丢弃过期结果；Web 本地对局按浏览器会话隔离，避免不同客户端共享棋盘。
- Web 状态增加 AI 错误与搜索统计；新增固定局面性能诊断脚本，并扩充战术局面、缓存、节点预算、剪枝一致性、异步取消、会话隔离和真实棋盘恢复测试。
- 将应用元数据和 PWA 静态资源缓存版本统一更新为 0.3.1。

## [0.3.0] - 2026-07-16

- 新增“普通”难度 NormalAI，使用迭代加深 Negamax、Alpha-Beta 剪枝、增量 Zobrist 哈希、容量受限置换表和固定时间预算；不依赖强化学习、神经网络或外部服务。
- 新增带坐标去重的棋型识别、静态评估和威胁优先候选点生成，覆盖连续及带间隔的四、三、二、边界封堵和交叉双重威胁；一步获胜与必须阻挡的落点不受候选数量限制。
- 将普通难度接入 Pygame 和 Web 本地人机模式，重开后保留难度选择；简单难度继续使用原有 SimpleAI，困难难度继续显示为开发中。
- Web API 新增 AI 难度与思考状态，并在 AI 搜索期间阻止并发落子；私人双人联机房间保持原有无 AI 流程。
- 增加 SearchPosition 落子/撤销、哈希恢复、置换表边界、棋型、候选排序、搜索超时、剪枝一致性、可复现性及两套界面接入测试。
- 将应用元数据和 PWA 静态资源缓存版本统一更新为 0.3.0。

## [0.2.3] - 2026-07-15

- 调整了 Web 棋盘绘制方式，使本地版、Web 本地对局和联机房间统一使用 15×15 可落子棋盘。
- 增加了 Web 棋盘的九个加粗星位标记，并让最外圈落子点直接对齐完整棋盘边缘，不再显示多余半格扩展。
- 修复了 Quick Tunnel 启动时端口已被占用却误报“公网对战已就绪”的问题，改为启动前检查端口并给出明确提示。
- 修复了 Quick Tunnel 继承 Named Tunnel Token、可能将敏感凭据输出到日志的问题；临时公网启动时会主动移除相关环境变量。
- 调整了 Web 静态资源缓存版本，确保棋盘布局更新后浏览器和 PWA 能加载最新样式与脚本。

## [0.2.2] - 2026-07-15

- 增加了 SimpleAI 同优先级落点的交叉处理：一步可同时扩展或阻挡两条连续棋线时优先选择该位置。
- 调整了单子八邻格的选择顺序：优先向棋盘中心靠近，距离相同时依次选择横、竖、斜方向。

## [0.2.1] - 2026-07-14

- 调整了 SimpleAI 的连续棋串选择：同长度、同一攻防类别下，两端都可落的棋串优先于单端可落的棋串。
- 修复了连续三中先扫描到单端棋串时，可能忽略后续两端均可落棋串的问题。

## [0.2.0] - 2026-07-14

- 调整了 SimpleAI 的单子策略：不再随机落子，优先在己方孤立棋子周围八格扩展为二。
- 增加了己方单子无法扩展时，在对方孤立棋子周围八格落子阻挡的策略。
- 修复了仅存在单子时 SimpleAI 可能在远处随机落子的行为。

## [0.1.5] - 2026-07-14

- 增加了 Quick Tunnel 一键启动脚本，自动获取临时 HTTPS 公网地址并以该地址生成房间链接与二维码。
- 调整了临时跨网对战流程，启动时只需运行一条命令，无需手动在两个终端间传递公网地址。
- 修复了从 `127.0.0.1` 页面创建房间时生成本机邀请链接、手机无法访问的问题；现在会直接阻止并提示正确启动方式。

## [0.1.4] - 2026-07-14

- 增加了房主邀请弹窗、本机生成的二维码、浏览器内邀请链接保存和手动重连入口。
- 增加了本地与房间对局的结束弹窗、双方累计用时展示、再来一局操作和本地重新开始二次确认。
- 调整了首页，将私人远程对战作为清晰的主入口；AI 难度仅在人机模式下显示。
- 调整了手机端棋盘体验，加入横屏提示、落子触摸预览、中文无障碍标签和键盘焦点样式。
- 增加了 PWA 清单、图标和离线页面缓存，可添加到手机主屏幕使用。
- 修复了二维码依赖第三方服务可能泄露私人邀请链接的问题，改为服务端本机生成且不缓存响应。

## [0.1.3] - 2026-07-14

- 私人房间邀请密钥改存于 URL 片段，并改为 WebSocket 首条消息认证，避免密钥出现在页面请求和 WebSocket 握手日志中。
- 增加创建房间的频率与总量限制，默认关闭公网 API 文档，并加入后台过期房间清理。
- 每个房主/受邀者座位限制为单连接；断线时自动暂停并结算累计用时，双方重连后自动恢复。

## [0.1.2] - 2026-07-14

- 验证并记录 Cloudflare Quick Tunnel 的临时跨网对战流程：分别运行本机服务和 Tunnel，使用随机 HTTPS 邀请地址在不同 WLAN 设备间对战。
- 重构网络对战文档，区分临时 Quick Tunnel、同一 Wi-Fi 局域网和长期固定域名三种启动方式，并说明各自的访问边界。

## [0.1.1] - 2026-07-14

- 新增固定公网地址的 Cloudflare Tunnel 启动器：Mac 与手机可通过同一 HTTPS/WSS 地址进行私人房间对战，无需处于同一 WLAN 或开放路由器端口。

## [0.1.0] - 2026-07-13

- 新增基于 WebSocket 的私有双人房间：邀请链接、受邀者选择棋色和先手、房主确认开局、实时同步，以及双方确认的悔棋和再来一局。
- 增加 Docker 运行配置与公网 HTTPS/WSS 部署说明。

## [0.0.0] - 2026-07-13

### Added

- 15×15 五子棋核心：合法落子、轮流行棋、胜负/平局判定、重开与悔棋。
- Pygame 本地双人和人机对战，以及浏览器版双人和人机对战。
- SimpleAI：按连续四、三、二的优先级进攻或防守；单子时随机落子。
- 获胜棋线高亮、黑白双方累计用时，以及开始对局后启动的计时逻辑。
- 鼠标按钮式的模式、AI 难度、开始、重开和悔棋操作；当前启用简单难度，普通和困难预留开发。
- 覆盖核心规则、AI、计时、服务端接口和 Web 状态的自动化测试。
